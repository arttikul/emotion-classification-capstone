# Emotion Classification - Deep Learning Capstone

6-класовий класифікатор емоцій у тексті, побудований як повний цикл розробки:
EDA → класичний базовий рівень → модель глибокого навчання з нуля →
дообучений трансформер → порівняння та аналіз.

## Задача та датасет

**Задача:** класифікувати коротке речення англійською мовою на одну з шести
емоцій: `sadness`, `joy`, `love`, `anger`, `fear`, `surprise`.

**Датасет:** [dair-ai/emotion](https://huggingface.co/datasets/dair-ai/emotion)
(нерозділена версія), ~417 000 розмічених речень, зібраних із твітів. Дві
колонки: `text` (сире речення) та `label` (ціле число 0–5). Розподіл класів
незбалансований - `joy` та `sadness` домінують, `surprise` та `love` рідкісні -
це вирішується за допомогою стратифікованого розбиття та зваженої за класами
функції втрат.

Розмістіть CSV-файл за шляхом `data/emotion-dataset.csv` перед запуском (не
додається до git через розмір - див. `.gitignore`).

## Підхід

Три моделі навчаються та порівнюються на одному й тому ж стратифікованому
розбитті 70/15/15 train/val/test, щоб порівняння було коректним:

| # | Модель | Що демонструє |
|---|-------|----------------------|
| 1 | **TF-IDF + Logistic Regression** | Класичний базовий рівень ML / перевірка адекватності |
| 2 | **BiLSTM** (PyTorch, навчається з випадкової ініціалізації) | Основи глибокого навчання: ембединги, рекурентне моделювання послідовностей, регуляризація |
| 3 | **Дообучений DistilBERT** (Hugging Face `transformers`) | Transfer learning із попередньо навченим трансформером |

Кожна модель оцінюється за accuracy та **macro-F1** (саме macro, а не micro,
через дисбаланс класів - модель, яка добре розпізнає лише `joy`/`sadness`,
не повинна виглядати штучно сильною).

## Як запустити (Google Colab, GPU runtime)

1. Відкрийте `notebook/emotion_classification.ipynb` у Colab (або завантажте
   всю папку репозиторію / клонуйте через `!git clone`).
2. Runtime → Change runtime type → **GPU** (T4 достатньо).
3. Завантажте `emotion-dataset.csv` у `data/` (або підключіть Google Drive).
4. Виконайте всі клітинки по порядку зверху вниз. Ноутбук:
   - завантажує та досліджує дані,
   - навчає та оцінює базовий рівень TF-IDF,
   - навчає та оцінює BiLSTM,
   - дообучує та оцінює DistilBERT,
   - будує порівняльну таблицю та графік поруч,
   - виконує інференс на кількох нових прикладах речень.

Для швидкого «smoke test» перед повним запуском встановіть `SAMPLE_FRAC`
(на початку ноутбука) у невелике значення, наприклад `0.05`.

### Локальний запуск замість Colab

```bash
pip install -r requirements.txt
python src/train_baseline.py --csv_path data/emotion-dataset.csv
python src/train_lstm.py --csv_path data/emotion-dataset.csv --epochs 6
python src/train_transformer.py --csv_path data/emotion-dataset.csv --epochs 2
```
Для останніх двох кроків настійно рекомендується GPU; дообучення DistilBERT
на повних даних лише на CPU практично нездійсненне.

## Що було перевірено перед передачею

Кожен модуль був написаний і протестований end-to-end на підмножині даних у
пісочниці лише з CPU (без GPU, з обмеженим мережевим доступом до Hugging Face
Hub):

- `src/data.py` - завантаження, очищення, стратифіковане розбиття: перевірено
  на повному CSV із 417 тис. рядків.
- `src/vocab.py` - побудова словника та кодування: покрито unit-тестами
  окремо.
- `src/train_baseline.py` - повний запуск на 5% стратифікованій вибірці:
  **92% accuracy / 0.89 macro-F1**.
- `src/models.py`, `src/train_lstm.py` - архітектуру та цикл навчання
  перевірено на синтаксичну коректність; PyTorch не вдалося встановити в
  пісочниці (CPU-збірка там недоступна - лише CUDA-збірка, яка не вміщується
  в дисковій квоті пісочниці), тому прямий/навчальний прохід потрібно
  запускати в Colab.
- `src/train_transformer.py` - перевірено на синтаксичну коректність;
  завантаження токенізатора/моделі з `huggingface.co` заблоковано мережевою
  політикою пісочниці, тому це також потребує Colab (де є повний доступ до
  інтернету) для першого реального запуску.
- `src/viz.py` - усі три функції побудови графіків успішно виконуються на
  тестових даних.

Коротко: усе, що можна було перевірити без GPU чи доступу до Hub, було
перевірено. Частини, що потребують Colab, використовують стандартні,
добре задокументовані API Hugging Face / PyTorch, але перед презентацією
варто самостійно один раз запустити повний ноутбук, щоб виявити будь-які
особливості середовища та заповнити фактичну таблицю результатів вище.
